# Meetlog

リアル会議をする社会人向けの、話者識別付き音声文字起こしWebアプリ。

## 構成

- `client/` — React (Vite)。開発時は `/api` を vite proxy で server へ転送
- `server/` — Express。`app.js` がアプリ本体、`index.js` はローカル起動用エントリ
- `api/index.js` — Vercel Functions エントリ（`server/app.js` をエクスポート）
- 依存はルート `package.json` に統合（client のみ個別の package.json）

## コマンド

- `npm run dev` — server(3000) + client(5173) を一括起動
- `npm test` — サーバー側ロジックのテスト（node:test）
- `npm --prefix client run build` — クライアントの本番ビルド

## 常時ルール

- 作業前に既存コードの問題点を確認する。修正とあわせてリファクタリングも行うが、修正とリファクタリングは同一コミットに混ぜず、分けてコミットする
- 機能追加の前後で git commit する。新機能は🆕付きで報告し、節目ごとに実装済み機能一覧を報告する
- ゆうの手動操作は最小限にする。手動操作が必要な場合は、画面遷移込みの手順をまとめて提示し、細切れに確認を求めない
- ユーザー向けの注意書きは、必要だと判断したら随時UI（画面下部・プレーンテキスト）に追加していく。ただし文面は実装前に必ずゆうの承認を取る
- UIの注意書き・ガイダンスは画面下部にプレーンテキストで表示。LP化・過剰デザイン禁止
- 作業のたびにコードのノイズ除去（不要コード・重複・整理）を行う
- AssemblyAIは「確定した選択」ではなく「差し替え可能なデフォルト」。STT呼び出しは必ず抽象化レイヤー（`server/stt/`）経由で行い、プロバイダー固有のコードをエンドポイントに直書きしない
- 要件が現行のツール・ライブラリ・外部サービスの限界にぶつかった場合、その制約内での回避策だけでなく、代替手段・代替ベンダーの調査結果を併せて提示する。設計の節目では現行の選定そのものを一度疑う
- 録音音声・会議データをサーバーやストレージに残さない方針を維持する（文字起こし完了後に即削除）。この方針に反する実装が必要になる場合は、実装前にゆうに相談する
