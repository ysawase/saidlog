# SaidLog UI・UX違和感ログ

## 目的

- SaidLog固有のUI・UX上の違和感、未決定案、実測結果を一元管理する
- ユーザー指摘とAI提案を混同しない
- Claude・GPT等のAIメモリだけへ依存しない
- 正式バックログや証拠カードとの二重管理を防ぐ

## 正本

このファイルをSaidLogのUI・UX違和感ログの正本とする。Claude・GPT等のメモリやチャット内の記述は、正本への反映が完了するまで一時情報として扱う。

## 項目書式

ID｜記録日｜分類｜起点｜内容｜状態｜確認状態｜出典｜移管先

起点の固定値：ユーザー直接指摘／AI提案／実測で判明／外部情報／出典不明
状態の固定値：未整理／検討中／採用／却下／撤回／バックログ移管済み／解決済み
確認状態の固定値：原文確認済み／ユーザー再確認済み／要ユーザー確認／実測証拠あり／外部出典確認済み／未確認

## 運用ルール

- ユーザー指摘、AI提案、実測結果を混同しない
- AIメモリから復元した内容だけを根拠に「ユーザー直接指摘」と確定しない
- 原文またはユーザー再確認がない場合は、確認状態を「要ユーザー確認」とする
- IDは一度採番したら再利用しない
- 採用、却下、撤回、移管、解決後も行を削除せず状態を更新する
- 正式バックログや証拠カードへ移管後は、詳細を二重記載せず移管先を残す
- 未整理候補をAIが勝手に分割、統合、確定しない
- ファイル更新は、ユーザーの依頼または許可を得て行う
- 新チャットでは現在の作業に関係する未整理、検討中、要ユーザー確認の項目だけを確認する
