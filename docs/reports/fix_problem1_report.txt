## 権限リクエストの実装方法

  start() 冒頭で requestPermissions() を呼び出し、
  返り値の recordAudio フィールドが 'granted' かどうかを確認する。
  'granted' でない場合（'denied' / 'prompt' / 'prompt-with-rationale'）は
  NotAllowedError をthrowし、Recorder.jsx のエラー表示分岐（t('recorder.errorMic')）に乗せる。

  requestPermissions() は内部でシステムダイアログを表示する（初回のみ）ため、
  ユーザーが許可/拒否した直後の状態を返す。

## stop() 戻り値変換の実装方法

  uri→Blob変換:
    Capacitor.convertFileSrc(result.uri) でネイティブURIをWebView参照可能なURLに変換し、
    fetch() → response.blob() で Blob を取得する。
    Capacitor は既存インポート（import { Capacitor } from '@capacitor/core'）があり追加不要だった。

  duration の単位:
    型定義（definitions.d.ts）の JSDoc に "The duration of the recording in milliseconds" と
    明記されていることを確認。秒→ミリ秒変換は不要で、result.duration をそのまま durationMs に使用。
    null/undefined の場合は ?? 0 でフォールバック。

  mimeType のハードコード:
    Android/iOS（uri ルート）: 'audio/mp4' をハードコード
    Web（blob ルート）: 'audio/webm' をハードコード
    ★ 検証ポイント: 実機確認で「文字起こし」ボタン押下後、Vercel上の文字起こしAPIが
       audio/mp4 を受理するか確認すること。
       拒否される場合は mimeType の調整またはサーバー側の受け入れ形式確認が必要。
