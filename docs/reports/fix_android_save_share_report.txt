## downloadBlob() の分岐実装の要約

recorder.js に以下を追加：
- インポート: Capacitor(@capacitor/core), Filesystem/Directory(@capacitor/filesystem), Share(@capacitor/share)
- blobToBase64() ヘルパー関数（FileReader で DataURL → base64文字列を取得）
- downloadBlob() を async 化し、Capacitor.isNativePlatform() で分岐：
  - Native時: blob → base64 → Filesystem.writeFile(Directory.Cache) → Share.share({files:[uri]})
    エラー時は alert('保存に失敗しました') のみ（ファイルパス等はログ出力しない）
  - Web時: 既存の URL.createObjectURL + <a download> + click() を維持
- 呼び出し元（Recorder.jsx / App.jsx）は変更なし（awaitなしでも動作に支障なし）

## export.js の修正内容

- triggerDownload(url, filename) を削除
- Capacitor / downloadBlob を import
- exportTxt / exportDocx: triggerDownload(URL.createObjectURL(blob), filename)
  → await downloadBlob(blob, filename) に統一（Blob直接渡し）
- exportPdf:
  - Web時: pdfMake.createPdf(docDef).download(filename) — 変更なし
  - Native時: pdfMake.createPdf(docDef).getBlob(cb) でBlob取得後、downloadBlob(blob, filename)

## Web版PDFエクスポートの既存動作確認
- ファイル名: formatFilename('pdf') を事前に計算して両分岐に同一値を渡しているため変化なし ✅
- 保存タイミング: else { pdfMake.createPdf(docDef).download(filename) } がそのまま実行されるため変化なし ✅
- エラー時の挙動: pdfmakeの.download()内部処理は変更していないため変化なし ✅
