■ レビューで発見・追加修正した不整合
================================================================

  S08-E（showCopyModal ログイン済み分岐）で plus_active 分岐が漏れていた。
  初期実装では web / loading / デフォルト の3分岐のみで、
  Plus 利用中ユーザー（plus_active）がモーダルを開いた場合に
  デフォルト分岐へ落ちて「SaidLog Plusに進む」ボタンが表示される
  バグ（Plus利用中への再購入CTA表示）が残存していた。

  最終コードレビュー時に発見し、先頭に plus_active 分岐を追加して修正。
  → 4分岐（plus_active / web / loading / デフォルト）に変更。

■ purchaseTake 呼び出しの集約
================================================================

  変更前: HistoryList.jsx / TranscriptView.jsx（2箇所）が
          purchaseTake を直接 import して呼び出し。
          未ログイン・Web 環境でのガードなし、エラー catch なし。

  変更後: purchaseTake の呼び出しは App.jsx の handleUpgrade 内1箇所のみ。
          - upgradeMode === 'purchase'（Native + ログイン済み + 無料プラン）の場合のみ呼ぶ
          - try/catch で購入失敗時のエラーハンドリングあり
          - 購入成功後に getAccountStatus() で accountStatus を更新

  HistoryList / TranscriptView は onUpgrade prop 経由で
  handleUpgrade を呼ぶだけ（billing.js を知らない設計）。
